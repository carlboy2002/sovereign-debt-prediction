## Project Overview

We build a multimodal early warning framework for sovereign debt crises that bypasses official macroeconomic statistics (which are released with lags and frequently manipulated). The framework combines three data streams:

- **NASA nighttime satellite imagery** (VIIRS) — proxy for true economic activity
- **GDELT global news database** — political and economic sentiment
- **World Bank macroeconomic indicators** — standard baseline features

We train nested XGBoost classifiers grounded in a two-period structural signaling game of government misreporting. The best model achieves **AUC 0.902** on the held-out validation set; adding news text features lifts average precision by **+14.8%** over macro-only baselines.

See `sovereign_debt_dashboard.html` for an interactive results dashboard.

## Getting Started (For New Teammates)

### 1. Clone the repository

```bash
git clone https://github.com/<your-username>/sovereign-debt-prediction.git
cd sovereign-debt-prediction
```

### 2. Install Python dependencies

Python 3.10+ required.

```bash
pip install -r requirements.txt
```

If `rasterio` or `fiona` fail to install, skip them — they're only needed for local satellite processing (we use Google Earth Engine instead):

```bash
pip install pandas numpy scikit-learn xgboost shap matplotlib seaborn statsmodels wbgapi google-cloud-bigquery google-cloud-bigquery-storage db-dtypes earthengine-api requests openpyxl tqdm
```

### 3. Set up Google Cloud (required for GDELT and Earth Engine)

This project needs a free Google Cloud account to access BigQuery and Earth Engine.

**a. Create a GCP project**
- Go to [console.cloud.google.com](https://console.cloud.google.com/)
- Click "New Project", name it anything (e.g., `sovereign-debt-yourname`)
- Note down your Project ID

**b. Enable required APIs**
- [BigQuery API](https://console.cloud.google.com/apis/library/bigquery.googleapis.com)
- [Earth Engine API](https://console.cloud.google.com/apis/library/earthengine.googleapis.com)

**c. Register for Earth Engine**
- Go to [earthengine.google.com](https://earthengine.google.com/) → Register
- Choose "Noncommercial / Academic" tier
- Link to your GCP project

**d. Authenticate locally**
```bash
gcloud auth application-default login
python auth.py    # triggers Earth Engine auth flow
```

### 4. Configure your local settings

```bash
cp config.example.py config.py
```

Open `config.py` and replace `"your-gcp-project-id-here"` with your actual GCP Project ID. **`config.py` is gitignored — never commit it.**

### 5. Run the pipeline

Run scripts in numbered order. Each step produces files used by the next.

**Phase 1 — Data acquisition** (one-time, ~30 min total)

```bash
# World Bank macro data (~1 min)
python scripts/01_fetch_wdi.py

# Nightlights: first export from Earth Engine (~20 min)
python fetch_nl_batch.py
# Wait for GEE tasks to complete at https://code.earthengine.google.com/tasks
# Download all 24 nl_YYYY.csv files from your Google Drive root
# Place them in data/raw/
python scripts/merge_nl_csvs.py
python scripts/02_fetch_nightlights.py

# GDELT news data (~3 min)
python scripts/03_fetch_gdelt.py

# Crisis labels (instant)
python scripts/04_fetch_crisis_labels.py
```

**Phase 2 — Processing** (~1 min)

```bash
python scripts/05_clean_merge.py
python scripts/06_feature_engineering.py
```

**Phase 3 — Analysis** (~5 min)

```bash
python scripts/07_descriptive_stats.py
python scripts/08_xgboost_model.py
python scripts/09_lasso_model.py
python scripts/10_fairness_audit.py
python scripts/11_structural_model.py
python scripts/12_misreporting_index.py
```

All outputs land in `outputs/figures/` and `outputs/tables/`.

## Known Issues and Future Work

**Data quality**
- The DMSP–VIIRS satellite sensor transition in 2013–2014 produces artifactual jumps in nightlight time series. Current workaround: VIIRS-only (2014+), which reduces sample size by ~40%.
- Proper fix: sensor harmonization following [Li et al. (2020)](https://doi.org/10.6084/m9.figshare.9828827). See `docs/TODO.md` for details.

**Methodology**
- LASSO convergence is marginal at the chosen regularization; elastic-net or PCA preprocessing may help.
- The originally planned DiD using VIIRS coverage expansion is infeasible (VIIRS had global coverage from day one). Alternative: use commodity price shocks as instruments.
- The misreporting index is exploratory; current ranking is biased toward small island / developed economies due to nightlight saturation.
